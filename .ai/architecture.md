# Architecture Notes

## Runtime Shape

`src/main.rs` initializes tracing and calls `awebpinator::app::run()`.

`src/app.rs` owns the Relm4 `Component`:

- `AppModel` stores application state.
- `AppMsg` represents UI/user actions.
- `CommandMsg` represents asynchronous command results for thumbnails, previews, and export.
- `AppWidgets` stores GTK widget handles that need to be synchronized during view updates.

The app uses a single main component rather than many smaller UI components. Keep changes local and consistent with the current message-driven style unless a larger refactor is explicitly requested.

## State Model

The core state lives in `AppModel`:

- `timeline: Timeline`: ordered frame list.
- `selection: BTreeSet<u64>` and `selection_anchor_id`: selected frames and Shift-click anchor.
- `clipboard: Vec<FrameItem>`: copied frames.
- `export_profile: ExportProfile`: current export settings.
- `diagnostics: Diagnostics`: ffmpeg/ffprobe availability state.
- `cache_dir: PathBuf`: temp cache for thumbnails and previews.
- `last_output_path: Option<PathBuf>`: selected export output path.
- `command_preview: String`: generated ffmpeg command preview.
- `preview_path` and `preview_frame_id`: selected-frame preview state.
- `drop_indicator_index`: timeline drag/drop target marker.
- `thumbnails_pending` and `export_in_progress`: background operation status.

Frame IDs are stable `u64` identifiers generated by `Timeline`. Selection should be based on IDs, not vector indexes.

## Data Types

`src/types.rs` contains shared serializable types:

- `FrameItem`: source image path, duration, transform, thumbnail path, enabled flag, and source dimensions.
- `TransformSpec`: rotation, flips, optional crop, optional resize, and fit mode.
- `ExportProfile`: export preset, dimensions, quality, lossless mode, encoder preset, compression settings, loop count, overwrite behavior, and raw ffmpeg args.
- `ProjectDocument`: serialized project state containing frames, export profile, and last output path.

Project files are JSON. Treat changes to these types as persistence-format changes and consider backward compatibility.

## Timeline And Selection

`src/timeline.rs` is the model layer for frame ordering and batch frame operations. It is covered by unit tests and should remain UI-independent.

`src/selection.rs` is UI-independent selection logic. It supports:

- Plain click: select one frame.
- Ctrl-click: toggle clicked frame.
- Shift-click: additive inclusive range from anchor.
- Ctrl-Shift-click: also additive inclusive range from anchor.

Keep selection behavior in `selection.rs` when possible instead of embedding special cases in GTK callbacks.

## Image Pipeline

`src/thumbnail.rs` handles local image processing with the `image` crate:

1. Open the source image.
2. Apply frame transforms in order: rotate, flip, crop, frame resize.
3. Optionally apply export resize and fit mode.
4. Save thumbnails, previews, or rendered export frames.

The cache directory is `${TMPDIR}/awebpinator-cache` via `std::env::temp_dir()`.

## Export Pipeline

`src/export.rs` handles WebP export:

1. Filter to enabled frames.
2. Create a temporary export directory.
3. Render each enabled frame to PNG.
4. Write an ffconcat manifest with per-frame durations and the required repeated final file.
5. Build an `ffmpeg` command using `libwebp_anim`.
6. Run system `ffmpeg`.
7. Return an `ExportJob` with temp dir, manifest path, output path, effective command, and status.

Advanced ffmpeg args are parsed with `shlex::split`. Invalid raw args should surface as command-preview/export errors, not panic.

## GTK UI Notes

The layout is built imperatively in `AppModel::init`:

- Header row: import/open/save and timeline edit actions.
- Main paned body: selected-frame preview on the left, diagnostics/selection edit/export panels on the right.
- Bottom timeline: horizontal thumbnail strip with drop slots.
- Footer status label.

Important current interaction rules:

- Timeline tiles show only thumbnail preview, frame number, and original filename.
- Selected tiles use a blue full-tile background.
- Reordering happens by dragging the tile itself.
- Ctrl+C, Ctrl+V, and Delete shortcuts are ignored while text-like widgets are focused.
- CSS is installed through `install_app_css(window)` after the GTK window/display exists.

## Async Work

Thumbnail, preview, and export work runs through Relm4 one-shot commands. When adding background work, update state through `CommandMsg` and keep GTK widget mutation inside `update_view`.
